# 简单电商项目构建过程手册

## 1. 项目概述

本项目是一个基于Spring Boot后端和Vue前端的简单电商应用。后端提供RESTful API服务，前端负责用户界面展示和交互。

## 2. 环境要求

- JDK 8+（推荐JDK 11+）
- Maven 3.6+ 或 Gradle 6.8+
- Node.js 14+ 和 npm 6+
- MySQL 8.0+ 数据库

## 3. 后端构建步骤

### 3.1 检查后端目录结构和配置

后端项目位于 `simple-shop-backend/backend` 目录下，是一个标准的Spring Boot项目结构：

```
backend/
├── pom.xml
├── src/
│   ├── main/
│   │   ├── java/
│   │   └── resources/
│   │       ├── application.properties
│   │       └── sql/
│   └── test/
└── target/
```

配置文件 `application.properties` 包含了服务器端口、数据库连接、JPA配置等关键信息。

### 3.2 构建后端项目（跳过测试）

使用Maven构建后端项目，跳过测试阶段：

```bash
cd simple-shop-backend/backend
mvn clean package -DskipTests
```

构建成功后，将在 `target` 目录下生成可执行的JAR文件：`simple-shop-0.0.1-SNAPSHOT.jar`

## 4. 前端构建步骤

### 4.1 检查前端目录结构和配置

前端项目位于 `simple-shop-frontend/frontend` 目录下，是一个标准的Vue项目结构：

```
frontend/
├── src/
│   ├── components/
│   ├── router/
│   ├── views/
│   ├── App.vue
│   └── main.js
├── babel.config.js
├── package.json
├── package-lock.json
└── vue.config.js
```

配置文件 `vue.config.js` 包含了开发服务器配置和API代理设置。

### 4.2 构建前端项目

首先安装项目依赖：

```bash
cd simple-shop-frontend/frontend
npm install
```

安装完成后，可以运行开发服务器：

```bash
npm run serve
```

## 5. 运行验证步骤

### 5.1 启动后端服务

```bash
cd simple-shop-backend/backend
java -jar target/simple-shop-0.0.1-SNAPSHOT.jar
```

后端服务将在 http://localhost:8081 上运行。

### 5.2 验证后端服务

使用curl命令检查后端服务健康状态：

```bash
curl http://localhost:8081/actuator/health
```

如果返回状态为"UP"，表示后端服务正常运行。

### 5.3 启动前端服务

```bash
cd simple-shop-frontend/frontend
npm run serve
```

前端服务将在 http://localhost:8080 上运行。

### 5.4 验证前端服务

在浏览器中访问 http://localhost:8080，应该能看到前端应用的登录页面或首页。

## 6. 常见问题与解决方案

### 6.1 后端构建失败

- 检查Java和Maven版本是否符合要求
- 确保数据库连接配置正确
- 检查依赖项是否存在冲突

### 6.2 前端构建失败

- 检查Node.js和npm版本是否符合要求
- 删除 `node_modules` 目录后重新安装依赖
- 检查网络连接，确保能正常下载依赖包

### 6.3 服务启动失败

- 检查端口是否被占用
- 确保数据库服务正在运行
- 检查配置文件中的路径是否正确

## 7. 项目访问地址

- 前端应用：http://localhost:8080
- 后端API：http://localhost:8081
- 后端健康检查：http://localhost:8081/actuator/health

## 8. 注意事项

1. 首次运行时，后端会自动创建数据库表结构
2. 确保上传目录 `uploads/images` 存在且有写入权限
3. 开发环境下，前端通过代理访问后端API，生产环境需要调整配置
4. 生产环境部署时，建议使用Nginx反向代理前端和后端服务
